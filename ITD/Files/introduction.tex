\subsection{Document Structure}
The document is organized into the following sections:
\begin{itemize}
    \item[] \textbf{Introduction:} The first section of this document, containing the Scope of the document, the definitions and acronyms, revision history and reference documents.

    \item[] \textbf{Product Functions:} This section describes the implemented functions, list the discarded functions that were excluded from the final implementation, and details other structural changes.

    \item[] \textbf{System Development Frameworks:} This section lists the technologies used to develop the application, including adopted frameworks, programming languages, middlewares, and external APIs.

    \item[] \textbf{Source Code Structure:} This section explains the organization of the source code directories.

    \item[] \textbf{Testing Activity:} This section details the testing strategy.

    \item[] \textbf{Installation Guide:} This section provides a step-by-step instructions to build and run both the backend and frontend components of the system.

    \item[] \textbf{Effort Spent:} This final section presents a breakdown of the hours spent by each group member for writing this document.
\end{itemize}
